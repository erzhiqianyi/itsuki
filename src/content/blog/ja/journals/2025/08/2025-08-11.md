## Visual Studio Code
今日は仕事のツールについて話そうと思います。今日の話題は Visual Studio Code(ビジュアルスダジオコード)

私は最近、Visual Studio Code を使い始めました。このブログはC Visual Studio Codedで書きました。文章を書くなら、軽いソフトで、複雑な機能は必要ありません。とても便利です。以前はGitHubのWebサイトで書いていたので、あまり便利ではありませんでした。

私は Visual Sutio Code を開発ツールとして使ってみたいと思い始めました。

最近は[JapaneseDictionary](https://github.com/erzhiqianyi/JapaneseDictionary)を作成しています。少しコードを書いていて、Pythonのコードです。簡単プロジェクトです。

そして、Visual Studio Codeでプロジェクトを作成し、Pythonの開発環境を準備しました。ここまでは簡単でした。AIの力を借りて、自然言語で環境を構築することができました。私はそれを楽な仕事だと考えております。

しかし、コードを書いたりデバッグをしたりするのは、そんなに楽ではありません。

私は以前はWindowsを使っていて、今仕事のコンビューターもWindowsです。開発ツールはIntelliJ製品を使っています。Javaの開発にはIntelliJ IDEAを使い、PythonにはPycCharmを使っています。ドキュメントは時々IntelliJで書いています。

今、私のノートパソコンはMacです。おそらく4年間使っています。

恥ずかしいですね、この4年間で、この Macではあまりコードを書きませんでした。

Visual Studio Code と言えば、まったく新しいツールで、使い勝手があまり良くなく、IntelliJのツールほど直感的ではありません。

さあ、人生は短いですから、慣れたツールを使いましょう。

Visual Studio Codeはこれまで文章を書くツールとして使ってきましたが、やはり開発はIntelliJの方が専門的ですね。
